# Claude Code Instructions

## Design Persistence Rule

Every time you propose a system design, architecture, or technical solution:

1. You MUST save the design into /docs/designs/
2. File name format:
   YYYY-MM-DD-[short-title].md

## Document Structure

Each design must include:

- Background
- Problem Statement
- Proposed Solution
- Alternatives
- Trade-offs
- Final Decision

## Important

- Do NOT keep important design only in chat
- Always persist important design into files
